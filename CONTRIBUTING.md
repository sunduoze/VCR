# Contributing to VCR

English | [中文](#贡献指南)

Thank you for your interest in contributing to VCR! This document provides guidelines and instructions for contributing.

## Code of Conduct

Please be respectful and considerate of others when contributing to this project.

## How to Contribute

### Reporting Bugs

- Use the GitHub Issues tracker
- Describe the bug in detail
- Include steps to reproduce
- Include expected vs actual behavior
- Include screenshots if applicable

### Suggesting Enhancements

- Use the GitHub Issues tracker
- Describe the enhancement in detail
- Explain why this enhancement would be useful

### Pull Requests

1. Fork the repository
2. Create a feature branch (git checkout -b feature/your-feature)
3. Commit your changes (git commit -am 'Add some feature')
4. Push to the branch (git push origin feature/your-feature)
5. Create a Pull Request

### Code Style

- **Dart**: Follow the [Dart Style Guide](https://dart.dev/guides/language/effective-dart/style)
- **Rust**: Follow the [Rust Style Guide](https://doc.rust-lang.org/1.0.0/style/README.html)
- Use meaningful variable names
- Write comments for complex logic
- Keep functions small and focused

### Testing

- Add unit tests for new features
- Add integration tests for complex workflows
- Ensure all tests pass before submitting PR

### Documentation

- Update README.md if needed
- Add doc comments to new functions/classes
- Update CHANGELOG.md with your changes

## Development Setup

See README.md for detailed setup instructions.

# 贡献指南

[English](#contributing-to-vcr) | 中文

感谢您对 VCR 项目的关注！本文档提供了贡献的指南和说明。

## 行为准则

在为本项目做出贡献时，请尊重并考虑他人。

## 如何贡献

### 报告 Bug

- 使用 GitHub Issues 跟踪器
- 详细描述 bug
- 包含重现步骤
- 包含预期行为 vs 实际行为
- 如适用，包含截图

### 建议增强功能

- 使用 GitHub Issues 跟踪器
- 详细描述增强功能
- 解释为什么这个增强功能有用

### Pull Requests

1. Fork 本仓库
2. 创建功能分支 (git checkout -b feature/your-feature)
3. 提交更改 (git commit -am 'Add some feature')
4. 推送到分支 (git push origin feature/your-feature)
5. 创建 Pull Request

### 代码风格

- **Dart**: 遵循 [Dart 风格指南](https://dart.dev/guides/language/effective-dart/style)
- **Rust**: 遵循 [Rust 风格指南](https://doc.rust-lang.org/1.0.0/style/README.html)
- 使用有意义的变量名
- 为复杂逻辑编写注释
- 保持函数小而专注

### 测试

- 为新功能添加单元测试
- 为复杂工作流添加集成测试
- 在提交 PR 前确保所有测试通过

### 文档

- 如需要，更新 README.md
- 为新函数/类添加文档注释
- 在 CHANGELOG.md 中记录您的更改

## 开发环境设置

有关详细的设置说明，请参阅 README.md。

**Thank you for contributing to VCR!**
